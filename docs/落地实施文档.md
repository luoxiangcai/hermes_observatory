## 2. 技术栈选型

| 层 | 选型 | 理由 |
|----|------|------|
| 后端框架 | **FastAPI** | 原生支持 async、WebSocket、自动 OpenAPI 文档；Hermes 生态用 Python |
| 前端框架 | **原生 JS + 轻量组件** | 不引入 React/Vue 等重框架；单页应用，原生 JS 足够；原型已是原生 JS |
| 图表库 | **Chart.js** 或 **ECharts** | 轻量、无需构建步骤、CDN 引入即可 |
| Diff 引擎 | **diff-match-patch** (Google) | 纯 Python，用于技能版本 diff |
| WebSocket | FastAPI 内置 | 无需额外依赖 |
| SQLite 读取 | Python 标准库 `sqlite3` | 无需额外依赖 |
| 文件监控 | `watchdog` | 监控 `~/.hermes/` 文件变化，触发采集器重读 |
| HTTP 客户端 | `httpx` | 调用 Hermes Dashboard API（`/api/status`、`/api/config/schema`） |
| 测试 | `pytest` | Hermes 生态标准 |
| 部署 | `uvicorn` + `systemd` 或 `docker` | 单进程，端口可配置 |

**关键约束**：
- 不引入 Node.js / npm 构建链——前端是纯 HTML+CSS+JS，CDN 引入 Chart.js
- 不引入数据库——所有数据从 Hermes 文件系统实时读取，观测台自身无状态
- 不引入消息队列——WebSocket 直连，采集器直接推送

## 3. 后端 API 规范

### 3.1 REST API

所有端点支持 `?profile=<name>` 查询参数，路由到对应 Profile 的 `HERMES_HOME`。

| 方法 | 路径 | 返回 | 说明 |
|------|------|------|------|
| GET | `/api/overview` | 总览统计数据 | 技能数、记忆使用率、进化次数、待审批数 |
| GET | `/api/timeline` | 事件列表 | 支持 `?limit=50&type=skill&profile=default` 过滤 |
| GET | `/api/memory` | 记忆内容 | MEMORY.md + USER.md 内容 + 容量 |
| GET | `/api/skills` | 技能列表 | 含 .usage.json 统计 + 状态 |
| GET | `/api/skills/{name}` | 技能详情 | 含版本历史 + diff |
| GET | `/api/skills/{name}/diff?from=v1&to=v2` | Diff 结果 | 两个版本之间的文本 diff |
| GET | `/api/curator/runs` | Curator 运行历史 | 含 run.json + REPORT.md |
| GET | `/api/gepa/runs` | GEPA 进化运行列表 | 含 metrics.json |
| GET | `/api/gepa/runs/{id}` | GEPA 运行详情 | 含 baseline vs evolved 分数 + diff |
| GET | `/api/pending` | 待审批列表 | memory + skills pending |
| POST | `/api/pending/{id}/approve` | 审批结果 | 转发给 Hermes 原生 API |
| POST | `/api/pending/{id}/reject` | 审批结果 | 转发给 Hermes 原生 API |
| GET | `/api/narrative` | 进化叙事 | 自动生成的故事线 Markdown |
| GET | `/api/profiles` | Profile 列表 | 所有可用 Profile |
| GET | `/api/drift/status` | 变化检测结果 | 当前 Drift 状态 |
| GET | `/api/drift/health` | 数据源健康检查 | 所有数据源状态 |

## 5. 采集器插件接口

```python
# backend/collectors/base.py

from abc import ABC, abstractmethod
from typing import Any
from pathlib import Path

class BaseCollector(ABC):
    """所有采集器的抽象基类。"""

    # 采集器元数据
    name: str           # 采集器名称，如 "memory"
    schema_key: str     # 对应 Schema 注册表中的 key，如 "memory_md"
    source_path: str    # 数据源路径模式，如 "~/.hermes/memories/MEMORY.md"
    poll_interval: int  # 轮询间隔（秒），0 = 仅启动时读取一次

    @abstractmethod
    def read(self, hermes_home: Path) -> Any:
        """从 Hermes 文件系统读取数据，返回标准化字典。"""
        pass

    @abstractmethod
    def validate(self, data: Any) -> tuple[bool, list[str]]:
        """校验数据是否符合已知 Schema。返回 (是否通过, 差异列表)。"""
        pass

    def on_drift(self, drifts: list[str]):
        """检测到 Schema Drift 时的回调。默认行为：记录日志。"""
        for d in drifts:
            logger.warning(f"[{self.name}] Schema drift: {d}")

    def to_event(self, data: Any) -> dict | None:
        """将读取的数据转换为进化事件（如果适用）。返回 None 表示不产生事件。"""
        return None
```

## 6. Plugin Hook 代码

### plugin.yaml

```yaml
name: hermes-observatory
description: Collect evolution events for the Hermes Observatory
version: 1.0.0
events:
  - post_tool
  - on_session_end
```

### handler.py

```python
"""Hermes Observatory — Plugin Hook Handler.

Observes tool calls related to evolution (memory, skill_manage, skill_view)
and writes structured events to evolution-events.jsonl.
"""
import json
import time
from pathlib import Path
from datetime import datetime, timezone

LOG_FILE = Path.home() / ".hermes" / "logs" / "evolution-events.jsonl"

# 需要观察的工具
EVOLUTION_TOOLS = {"memory", "skill_manage", "skill_view"}

# 需要观察的 skill_manage action
SKILL_ACTIONS = {"create", "patch", "edit", "delete", "write_file", "remove_file"}

# 需要观察的 memory action
MEMORY_ACTIONS = {"add", "replace", "remove"}


def post_tool(ctx, tool_name: str, args: dict, result: str):
    """Fire-and-forget observer. Does not block or modify the tool call."""
    if tool_name not in EVOLUTION_TOOLS:
        return

    action = args.get("action", "")
    event = _build_event(ctx, tool_name, action, args, result)
    if event:
        _write_event(event)


def on_session_end(ctx):
    """Check for background review fork output at session end."""
    event = {
        "event_id": f"evt_{int(time.time()*1000)}",
        "timestamp": datetime.now(timezone.utc).isoformat(),
        "profile": ctx.get("profile", "default"),
        "hermes_version": ctx.get("version", "unknown"),
        "event_type": "session_end",
        "tool": "session",
        "action": "end",
        "target": ctx.get("session_id", ""),
        "session_id": ctx.get("session_id", ""),
        "turn_number": ctx.get("turn_count", 0),
        "trigger": "session_end",
        "details": {}
    }
    _write_event(event)


def _build_event(ctx, tool_name, action, args, result):
    """Build a structured event from a tool call."""
    if tool_name == "skill_manage" and action in SKILL_ACTIONS:
        return {
            "event_id": f"evt_{int(time.time()*1000)}",
            "timestamp": datetime.now(timezone.utc).isoformat(),
            "profile": ctx.get("profile", "default"),
            "hermes_version": ctx.get("version", "unknown"),
            "event_type": f"skill_{action}",
            "tool": tool_name,
            "action": action,
            "target": args.get("name", ""),
            "target_type": "skill",
            "session_id": ctx.get("session_id", ""),
            "turn_number": ctx.get("turn_count", 0),
            "trigger": ctx.get("write_origin", "foreground_turn"),
            "details": {
                "diff_summary": _extract_diff_summary(action, args, result),
            }
        }

    if tool_name == "memory" and action in MEMORY_ACTIONS:
        return {
            "event_id": f"evt_{int(time.time()*1000)}",
            "timestamp": datetime.now(timezone.utc).isoformat(),
            "profile": ctx.get("profile", "default"),
            "hermes_version": ctx.get("version", "unknown"),
            "event_type": f"memory_{action}",
            "tool": tool_name,
            "action": action,
            "target": args.get("target", ""),
            "target_type": "memory_entry",
            "session_id": ctx.get("session_id", ""),
            "turn_number": ctx.get("turn_count", 0),
            "trigger": ctx.get("write_origin", "foreground_turn"),
            "details": {
                "content_preview": args.get("content", "")[:200],
            }
        }

    return None


def _extract_diff_summary(action, args, result):
    """Extract a short summary of what changed."""
    if action == "create":
        return f"Created skill: {args.get('name', '')}"
    if action == "patch":
        old = args.get("old_string", "")[:50]
        new = args.get("new_string", "")[:50]
        return f"Patched: '{old}...' → '{new}...'"
    if action == "delete":
        return f"Deleted skill: {args.get('name', '')}"
    return ""


def _write_event(event):
    """Append event to JSONL file. Non-blocking, errors are swallowed."""
    try:
        LOG_FILE.parent.mkdir(parents=True, exist_ok=True)
        with open(LOG_FILE, "a", encoding="utf-8") as f:
            f.write(json.dumps(event, ensure_ascii=False) + "\n")
    except Exception:
        pass  # Never crash the agent
```

## 11. 风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| Hermes 更新后文件路径变更 | 采集器读取失败 | Drift 探测告警 + 降级模式 + 采集器插件可热更新 |
| Plugin Hook 事件格式变更 | 事件丢失或格式错误 | Schema 注册表校验 + 自适应渲染 + 未知字段保留 |
| `~/.hermes/` 文件并发写入 | 读取到不完整数据 | 采集器用 `try/except` + 重试 + 文件锁检测 |
| 大量技能导致 git diff 慢 | 前端卡顿 | 分页加载 + 后端缓存 diff 结果 + 只 diff 最近 N 个版本 |
| 多 Profile 同时访问 | 数据混淆 | 每个 API 请求强制携带 `?profile=`，后端隔离 |
| Hermes Dashboard 未启动 | `/api/status` 不可用 | 降级为文件系统扫描获取版本号 |
| evolution-events.jsonl 过大 | 读取慢 | 按天轮转 + 只读最近 N 天 + 前端分页 |
| 观测台自身崩溃 | 用户无法查看进化状态 | 独立进程 + systemd 自动重启 + 不影响 Hermes Agent |

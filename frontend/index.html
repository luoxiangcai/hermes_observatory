🔌事件采集 — Plugin Hook 集成

通过 Hermes 的 Plugin Hook 系统实时捕获进化事件：

post_tool(memory) → 记忆更新事件
post_tool(skill_manage) → 技能变更事件
post_tool(skill_view) → 技能查看事件
on_session_end → 会话结束事件
post_llm_call → turn 计数
pre_verify → 验证门控事件

事件写入 ~/.hermes/logs/evolution-events.jsonl，通过 WebSocket 实时推送。

🛡️只读旁路原则

观测台是"只读旁路"的——它只采集、分析、呈现，不直接修改任何进化机制。

这样既不会引入新的故障面，也保证观测本身不影响被观测对象（像量子力学一样，观测者不扰动系统）。

✓ 只读文件系统（不写入 ~/.hermes/）
✓ Hook 是 fire-and-forget 观察者
✓ 审批操作转发给 Hermes 原生 API
✓ 独立进程，崩溃不影响 Agent
